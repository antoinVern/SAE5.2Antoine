# 🤝 Guide de Contribution

Merci de votre intérêt pour contribuer à ce projet ! 🎉

## 🚀 Comment contribuer

### Signaler un bug

1. Vérifiez que le bug n'a pas déjà été signalé dans les [Issues](https://github.com/antoinVern/SAE5.2Antoine/issues)
2. Créez une nouvelle issue avec :
   - Description claire du problème
   - Étapes pour reproduire
   - Comportement attendu vs comportement actuel
   - Version des outils utilisés (Vagrant, Ansible, VirtualBox)

### Proposer une amélioration

1. Créez une issue pour discuter de l'amélioration
2. Attendez la validation avant de coder
3. Suivez les conventions de code du projet

### Améliorer la documentation

Toute amélioration de la documentation est la bienvenue ! N'hésitez pas à :
- Corriger des erreurs
- Ajouter des exemples
- Clarifier des instructions

## 📝 Conventions

- Utilisez des messages de commit clairs et descriptifs
- Testez vos modifications avant de proposer une Pull Request
- Respectez la structure existante du projet

## 🔀 Processus de Pull Request

1. Fork le projet
2. Créez une branche pour votre fonctionnalité (`git checkout -b feature/ma-fonctionnalite`)
3. Committez vos changements (`git commit -m 'Ajout de ma fonctionnalité'`)
4. Push vers la branche (`git push origin feature/ma-fonctionnalite`)
5. Ouvrez une Pull Request

## ✅ Checklist avant de soumettre

- [ ] Code testé et fonctionnel
- [ ] Documentation mise à jour si nécessaire
- [ ] Pas d'erreurs de linting
- [ ] Messages de commit clairs

Merci pour votre contribution ! 🙏
